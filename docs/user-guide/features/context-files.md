# 컨텍스트 파일

컨텍스트 파일은 프로젝트별 지시사항과 설정을 Hermes Agent에게 전달하는 파일입니다.

## 지원 파일 유형

| 파일 | 설명 |
|------|------|
| `.hermes.md` / `HERMES.md` | Hermes 전용 프로젝트 컨텍스트 |
| `AGENTS.md` | 디렉토리별 에이전트 지시사항 |
| `CLAUDE.md` | Claude Code 호환 |
| `SOUL.md` | 성격 정의 (별도 로드) |
| `.cursorrules` | Cursor 호환 |
| `.cursor/rules/*.mdc` | Cursor 규칙 파일 |

## 우선순위

같은 유형의 파일이 여러 개 있으면 첫 번째로 발견된 파일만 로드됩니다. SOUL.md는 별도로 독립 로드됩니다.

## 로드 시점

### 시작 시

1. 현재 작업 디렉토리 스캔
2. UTF-8로 읽기
3. 보안 스캔 (프롬프트 인젝션 탐지)
4. 20,000자 초과 시 잘라내기
5. 시스템 프롬프트에 주입

### 세션 중

`SubdirectoryHintTracker`가 도구 호출에서 파일 경로를 감시하고, 새로운 하위 디렉토리의 AGENTS.md를 점진적으로 발견합니다.

## HERMES.md 예시

```markdown
# 프로젝트 컨텍스트

이 프로젝트는 Python FastAPI로 구축된 REST API입니다.

## 기술 스택
- Python 3.11, FastAPI, SQLAlchemy
- PostgreSQL, Redis
- Docker, GitHub Actions

## 코딩 규칙
- 타입 힌트 필수
- 테스트 커버리지 80% 이상
- Black으로 포매팅

## 주의사항
- `.env` 파일을 절대 커밋하지 마세요
- 마이그레이션은 항상 alembic으로
```

## AGENTS.md 점진적 발견

모노레포에서 디렉토리마다 다른 지시사항을 설정할 수 있습니다:

```
project/
├── HERMES.md          # 전체 프로젝트 컨텍스트
├── frontend/
│   └── AGENTS.md      # 프론트엔드 전용 지시사항
├── backend/
│   └── AGENTS.md      # 백엔드 전용 지시사항
└── infra/
    └── AGENTS.md      # 인프라 전용 지시사항
```

에이전트가 `frontend/` 하위 파일을 다루기 시작하면 해당 AGENTS.md가 자동으로 로드됩니다.

## 크기 제한

- **최대 20,000자**
- 초과 시 자동 잘라내기: 70% 앞부분 + 20% 뒷부분 + 10% 표시자

## 보안 스캔

컨텍스트 파일에서 다음 패턴을 감지합니다:

- **지시 우선순위 조작**: "이전 지시를 무시하라" 등
- **기만 패턴**: 다른 시스템인 척하기
- **자격증명 탈취**: API 키 요청 등
- **보이지 않는 문자**: 유니코드 zero-width 문자 등

## 베스트 프랙티스

1. **간결하게**: 꼭 필요한 정보만 포함
2. **구조화**: 마크다운 헤딩으로 섹션 구분
3. **모노레포**: 디렉토리별 AGENTS.md 활용
4. **버전 관리**: 컨텍스트 파일도 Git에 커밋
5. **민감 정보 제외**: 비밀번호, API 키 절대 포함하지 않기
