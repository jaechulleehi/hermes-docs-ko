# 영속 메모리

Hermes Agent는 세션 간에 지속되는 메모리 시스템을 제공합니다.

## 메모리 파일

| 파일 | 용량 제한 | 용도 |
|------|-----------|------|
| `MEMORY.md` | 2,200자 (~800 토큰) | 일반 지식, 프로젝트 정보 |
| `USER.md` | 1,375자 (~500 토큰) | 사용자 선호도, 스타일 |

파일 위치: `~/.hermes/memories/`

## 동작 방식

1. **자동 주입**: 세션 시작 시 두 파일이 시스템 프롬프트에 "냉동 스냅샷"으로 포함됩니다
2. **도구 기반 관리**: 에이전트가 메모리 도구로 내용을 추가/교체/삭제합니다
3. **읽기 불필요**: 메모리는 자동 주입되므로 별도의 읽기 도구가 없습니다

## 메모리 도구

| 동작 | 설명 |
|------|------|
| `memory_add` | 새 메모리 항목 추가 |
| `memory_replace` | 기존 항목 교체 |
| `memory_remove` | 항목 삭제 |

## 저장해야 할 것

- 사용자의 이름, 역할, 선호하는 작업 방식
- 프로젝트의 핵심 기술 스택, 패턴
- 반복적으로 사용하는 중요한 경로나 설정
- 사용자의 커뮤니케이션 스타일 선호

## 저장하지 말아야 할 것

- 코드나 git 히스토리에서 직접 확인 가능한 정보
- 임시적인 디버깅 정보
- 대화 컨텍스트에서 이미 명확한 정보
- 너무 세부적인 구현 디테일

## 용량 관리

메모리 용량에 도달하면 에이전트가 자동으로:
- 오래되거나 덜 중요한 항목 제거
- 유사한 항목 병합
- 중복 제거

## 세션 검색

SQLite FTS5 기반의 전문 검색으로 과거 세션을 검색할 수 있습니다:

```
> 지난주에 데이터베이스 마이그레이션에 대해 이야기한 적 있어?
```

Gemini Flash를 사용하여 관련 세션을 요약합니다.

## 외부 메모리 제공자

플러그인으로 외부 메모리 시스템을 연결할 수 있습니다:

| 제공자 | 설명 |
|--------|------|
| Honcho | 사용자 모델링 특화 |
| Mem0 | 범용 메모리 |
| Hindsight | 장기 기억 |
| RetainDB | 구조화된 메모리 |
| OpenViking | 벡터 기반 검색 |
| Holographic | 연관 기억 |
| ByteRover | 대용량 메모리 |
| Supermemory | 크로스 앱 메모리 |

## 보안

- 메모리 내용은 프롬프트 인젝션 스캔 대상
- 민감한 정보(비밀번호, API 키 등)는 저장하지 않도록 설계
