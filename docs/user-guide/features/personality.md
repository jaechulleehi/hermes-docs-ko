# 성격 및 SOUL.md

SOUL.md는 Hermes Agent의 성격을 정의하는 파일입니다. 시스템 프롬프트의 첫 번째 위치에 삽입됩니다.

## 파일 위치

```
~/.hermes/SOUL.md
```

## 역할

SOUL.md는 에이전트의 **목소리**를 정의합니다:
- 어조 (격식/비격식)
- 커뮤니케이션 스타일
- 직접성 수준
- 응답 길이 선호

## 기본 내용

첫 실행 시 자동으로 기본 SOUL.md가 생성됩니다. 한번 생성되면 자동으로 덮어쓰지 않습니다.

## 커스터마이징

```markdown
# SOUL.md

나는 한국어를 주로 사용하는 AI 어시스턴트입니다.

## 성격
- 친근하고 격식 없는 말투
- 기술적 설명은 정확하게
- 코드 예시를 적극 활용
- 불필요한 서론 없이 바로 본론

## 원칙
- 모르는 것은 모른다고 말하기
- 안전하지 않은 조언은 하지 않기
- 사용자의 의도를 먼저 이해하기
```

## 3가지 성격 레이어

| 레이어 | 파일 | 용도 | 변경 주기 |
|--------|------|------|-----------|
| SOUL.md | `~/.hermes/SOUL.md` | 안정적인 성격 특성 | 거의 안 바뀜 |
| AGENTS.md | 프로젝트별 | 프로젝트 특화 워크플로우 | 프로젝트별 |
| `/personality` | 세션 내 명령 | 임시 성격 오버레이 | 세션마다 |

## 내장 성격

Hermes에는 미리 정의된 성격이 포함되어 있습니다:

| 이름 | 설명 |
|------|------|
| `helpful` | 도움이 되는 (기본값) |
| `concise` | 간결한 |
| `technical` | 기술적인 |
| `creative` | 창의적인 |
| `teacher` | 교육적인 |

### 채팅에서 전환

```
/personality concise
/personality 한국어로 간결하게 대답해줘
```

## 커스텀 성격

`config.yaml`에서 커스텀 성격을 정의할 수 있습니다:

```yaml
personalities:
  korean_dev:
    description: "한국어 개발자 어시스턴트"
    content: |
      한국어로 대화합니다. 코딩 관련 질문에는
      항상 코드 예시와 함께 설명합니다.
```

## 보안

SOUL.md는 프롬프트 인젝션 스캔 대상입니다. 악의적인 내용이 감지되면 경고합니다.
