# 보안

Hermes Agent는 7중 심층 방어 모델을 채택합니다.

## 보안 계층

| 계층 | 설명 |
|------|------|
| 1. 사용자 인증 | 허용 목록, DM 페어링 |
| 2. 위험 명령 승인 | 위험한 명령 실행 전 사용자 확인 |
| 3. 컨테이너 격리 | Docker 보안 강화 |
| 4. MCP 자격증명 필터링 | 서브프로세스 환경 변수 필터링 |
| 5. 컨텍스트 파일 스캔 | 프롬프트 인젝션 탐지 |
| 6. 교차 세션 격리 | 세션 간 데이터 분리 |
| 7. 입력 정제 | 입력값 검증 및 정제 |

## 위험 명령 승인

### 승인 모드

| 모드 | 설명 |
|------|------|
| `manual` | 모든 위험 명령에 사용자 확인 요청 (기본값) |
| `smart` | AI 판단으로 안전한 명령은 자동 승인 |
| `off` | 승인 비활성화 (권장하지 않음) |

### YOLO 모드

```bash
hermes --yolo  # 모든 승인 우회 (주의!)
```

::: warning 주의
YOLO 모드는 모든 안전 확인을 우회합니다. 신뢰할 수 있는 환경에서만 사용하세요.
:::

### 위험 명령 패턴

다음과 같은 패턴이 감지되면 승인을 요청합니다:

- `rm -rf` — 재귀 삭제
- `chmod 777` — 과도한 권한 부여
- `DROP TABLE` — SQL 테이블 삭제
- `curl | bash` — 파이프를 통한 스크립트 실행
- 기타 위험 패턴

## 사용자 인증

### 허용 목록

```yaml
gateway:
  allowed_users:
    telegram: ["123456789"]
    discord: ["987654321"]
```

### DM 페어링

8자리 암호 코드를 사용한 인증:

```
사용자: /pair
봇: 페어링 코드: AbCd1234
사용자: AbCd1234
봇: 인증 완료!
```

### 속도 제한 및 잠금

- 연속 실패 시 자동 잠금
- 속도 제한으로 스팸 방지

## 컨테이너 격리

Docker 백엔드 사용 시 자동 적용되는 보안 강화:

```
--cap-drop ALL          # 모든 Linux 기능 제거
--security-opt no-new-privileges  # 권한 상승 차단
--pids-limit 100        # 프로세스 수 제한
--mount nosuid,noexec   # suid/exec 비활성화
```

## 자격증명 처리

- MCP 서브프로세스에 전달되는 환경 변수 필터링
- 오류 메시지에서 자격증명 자동 정제

## SSRF 보호

내부 네트워크 접근 차단:

- 사설 IP 대역 (10.x, 172.16.x, 192.168.x)
- 루프백 (127.0.0.1, localhost)
- 클라우드 메타데이터 엔드포인트 (169.254.169.254)

## 컨텍스트 파일 스캔

HERMES.md, AGENTS.md 등의 컨텍스트 파일에서 프롬프트 인젝션을 탐지합니다:

- 지시 우선순위 조작 시도
- 기만 패턴
- 자격증명 탈취 시도
- 보이지 않는 문자 (유니코드 조작)
